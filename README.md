## サービス概要

プログラミング学習者やエンジニアが技術タスクに取り組む際に、チャット形式で気軽にメモを取れるアプリです。

- タスクを ToDo リストで設定し、その下にチャット形式で気軽にメモが取れます。
- 気になった公式ドキュメントや技術記事をマイリストにカード形式で保存できます。
- タスクで参照したドキュメントや記事をコミュニティ内で共有できます。
- 完了したタスクの内容から TIL(Today I Learned)を生成し、GitHub の TIL リポジトリに成果を記録できます。

## このサービスへの思い・作りたい理由

新機能実装やバグ修正といった技術タスクに取り組む際、私はこれまで Notion や Arc ブラウザの Easel 機能を使ってメモを取っていましたが、それぞれに使いづらさを感じていました。
Notion はドキュメントとしての体裁を整える前提があるため、自然と文章化を意識してしまい、気軽に思考をメモするツールとしては向いていないと感じました。
一方、Arc の Easel 機能は、キャンバスにテキストやドキュメントの URL を貼り付ける方式で、気軽にメモを取れる点は良いものの、ドキュメントとメモの関係性が視覚的に把握しづらく、整理に手間取ることが多くありました。
こうした課題を解決するために、本サービスではチャット形式でメモやドキュメント、コードスニペットを投稿できるようにし、それらを階層化することで、タスクの解決プロセスを視覚的に整理・把握できるようにしています。

また、技術タスクの中で参照したドキュメントは、タスク完了後にはあまり見返されなくなる傾向にありました。
これは、例えばブラウザのブックマーク機能を使って保存していても、階層構造が複雑になりがちで、後から目的の情報にたどり着きにくくなることが一因です。
さらに、Arc のようにそもそもブックマーク機能がないブラウザを使っている場合は、ドキュメントの保存自体が難しくなります。
そこで、本サービスではドキュメントをアプリ内でカードとして保存し、タグをつけて管理できるようにすることで、後からタグ検索によって再発見しやすい設計とします。
こうすることで、タスク中に見つけた有益な情報を「一時的なもの」にせず、持続的な知識ストックとして活用できるようになります。

さらに、完了したタスクやそのメモを残しておくことで、将来的に同じような問題に直面した際、すぐに参照して解決へとつなげることができます。加えて、蓄積した情報をもとに、記事投稿や発表のネタとして再利用することも可能です。
単なる作業メモにとどまらず、自分の学びや経験をアウトプットにつなげるための基盤として、このサービスを活用してもらえることを目指しています。

## ユーザー層について

本アプリは、Web 系の技術に取り組むプログラミング学習者やエンジニア、特に初学者を主なターゲットとしています。
初学者は学習中に参照する情報が多く、思考や記録の整理に苦労しやすいため、本アプリの支援効果がより大きいと考えています。

具体的には、以下のようなユーザーを想定しています。

- タスクメモの整理に悩んでいる方  
  → チャット形式により、気軽に思考や記録を残すことができるため

- ブラウザのブックマーク機能では技術情報をうまく整理・保存できていない方  
  → ドキュメントや技術記事をカード形式でマイリストに保存し、タグ管理できるため

- Arc などのようにブックマーク機能を持たないブラウザを使用している方  
  → アプリ内で完結した保存・整理機能を提供し、ブックマーク機能を補完できるため

- TIL を書きたいと思っているが、続けるのが難しいと感じている方  
  → タスク完了時に TIL を自動生成し、GitHub の TIL リポジトリにそのまま記録できるため

- 他の学習者やエンジニアと情報共有・知見の相互活用をしたい方  
  → コミュニティ機能を通じて、技術情報の公開・共有が可能なため

## サービスの利用イメージ

### 技術タスクに取り組み、メモを取りたい時

- ユーザーはタスクページを新規作成するか、既にあるタスクページに移ります。
- タスクページでタスクの工程や疑問点などを ToDo リスト化します。
- ToDo リストの下にチャット形式で参照したドキュメントやコードスニペット、メモを投稿します。
- また、投稿したドキュメントはマイリストに追加したり、コミュニティページで共有できます。

### til リポジトリに学習成果を反映させたい時

- ユーザーはタスクを完了させるとページ内容から TIL 生成することを選択できます。
- TIL 生成を選択した場合は、GitHub 連携を行い、生成された TIL を GitHub の TIL リポジトリに push します。

### ドキュメント・技術記事をアプリ内で保存したい時

- ユーザーはマイリストページで気になったドキュメントや技術記事を保存できます。
- また、タスクページで自分が投稿した、あるいはコミュニティページで他のユーザーが投稿したドキュメントを右クリックすると、マイリストへ追加を選択できます。

### 情報共有・情報収集したい時

- ユーザーはタスクを完了させるとタスクページに投稿したドキュメントをコミュニティページに公開するかどうか選択できます。
- コミュニティページに投稿されたドキュメントはマイリストへ追加することができます。

## ユーザーの獲得について

本アプリは、プログラミングスクールに通う受講生や、プログラミングを学び始めた初学者、現役のエンジニアを主なターゲットとしています。
これらの層に向けて、スクール内での情報発信やコミュニティ活動を通じてサービスの存在を知ってもらい、ユーザーの獲得を図ります。

また、特定の機能に関心を持って利用を始めたユーザーが、他の関連機能にも興味を持ってもらうことで、より継続的に利用してもらえるような導線設計も意識しています。
たとえば、最初はドキュメント保存機能に魅力を感じて使い始めたユーザーが、TIL 生成機能の便利さに気づき、日々の記録にも活用するようになることで、定着率の向上を目指します。

## サービスの差別化ポイント・推しポイント

従来のタスク管理アプリでは、「気軽にメモを取れること」と「メモ同士の関係性やタスクの時系列を可視化すること」の両立が難しく、ユーザーはそのどちらかを妥協せざるを得ませんでした。
本アプリでは、チャット形式を採用することで、こうした両立の難しさを解消し、タスクに取り組むユーザーの思考と記録の流れを自然な形で整理できる点が特長です。

さらに、以下のような機能を備えることで、既存のタスク管理ツールとの差別化を図っています。

- 気になった公式ドキュメントや技術記事を、カード形式でマイリストに保存し、後からの参照や整理がしやすくなっています。
- タスクページで投稿したドキュメントや記事を、他のユーザーとコミュニティ内で共有することで、知識の相互活用が可能です。
- 完了したタスクの内容をもとに TIL を自動生成し、GitHub の TIL リポジトリに記録することで、学びの可視化と継続的なアウトプットが実現できます。

## 主な機能一覧

### MVP リリースで実装予定の機能

#### TOP 画面

- サービス内容を説明

#### タスク一覧画面

- タスクページの一覧表示（作成順）
  - タスクのステータス（「検討中」、「着手中」、「完了」）ごとに一覧表示するタスクを変更
- 新規作成機能
  - タスクページへのタグづけ(gem `acts-as-taggable-on`)

#### タスク詳細画面

- ToDo リストの作成
  - タスクの工程や疑問点をリスト化
- チャット形式の投稿機能
  - ドキュメント(URL)・コードスニペット・メモを投稿できる
  - 投稿時のリアルタイム更新(gem `hotwire-rails`)
  - 投稿に対して備考を投稿するような階層的なチャット投稿機能、投稿の親子関係を管理(gem `ancestry`)

#### ユーザー新規登録画面・ログイン画面

- ユーザー認証(gem `devise`)

#### マイリスト画面

- 保存したドキュメント・技術記事の一覧表示（保存順）
- ドキュメント投稿機能
  - ドキュメント・技術記事の URL を投稿する
  - ドキュメント・技術記事へのタグづけ(gem `acts-as-taggable-on`)
- キーワード検索機能(gem `ransack`)
- ページネーション機能(gem `kaminari`)

### 本リリースまでに実現したい機能

#### タスク一覧画面

- タスクページのキーワード・タグ検索機能(gem `ransack`)
- ページネーション機能(gem `kaminari`)

#### タスク詳細画面

- チャット形式の投稿機能
  - ドキュメントは URL から OGP 情報を取得し、カード形式で表示する(gem `nokogiri`)
  - コードスニペットは Markdown 形式で投稿すると、シンタックスハイライト表示する(gem `rouge`)
- コミュニティ共有機能

  - 完了ボタンを押すと、タスクページ内に投稿したドキュメントの内、コミュニティ画面で共有するドキュメントを選択できる。

- GitHub のリポジトリに TIL を記録する機能（TIL 機能 step.1）
  - GitHub の TIL リポジトリと連携する機能(gem `octokit`)
  - Markdown 形式で TIL を編集する機能
  - 編集した TIL を TIL リポジトリにプッシュする機能(gem `octokit`)

#### マイリスト画面

- ドキュメント・技術記事のタグ検索機能(gem `ransack`)

#### コミュニティ画面

- 投稿されたドキュメント・技術記事の一覧表示（投稿順）
- マイリスト保存機能
- キーワード検索機能(gem `ransack`)
- ページネーション機能(gem `kaminari`)

- ドキュメント・技術記事のタグ検索機能(gem `ransack`)
- 一覧表示について、マイリスト数順を追加

### 本リリース後に追加したい機能

#### タスク詳細画面

- TIL テンプレート生成機能(TIL 機能 step.2)
  - TIL 編集画面にルールベースで生成した TIL をプリセット
- TIL 生成機能（TIL 機能 step.3）
  - TIL 生成ボタンを押すと、タスクページの内容から、OpenAI API を利用し TIL を生成する
    - 入力: タスクページのタイトル・タグ・ToDo リストの内容、ページ内に投稿したドキュメント・コードスニペット・メモの内容
    - 出力: TIL(Markdown 形式)
    - ドキュメントの URL から内容を取得し、タイトルや説明文を抽出する(open_uri, gem `nokogiri`)
    - 投稿内容は親子関係を考慮する(gem `ancestry`)
    - OpenAI API の利用制限を設け、一回あたりのトークン数（特に入力）を制限することで利用コストを削減する

#### ユーザー新規登録画面・ログイン画面

- GitHub 認証(gem `omniauth-github`)
